# 월급쟁이부자들 과제 도메인

- 회원 가입 및 강의를 등록하고 신청할 수 있는 애플리케이션
- 다음과 같은 내용이 이 과제에서 요구하는 포인트
  - 서비스 요구사항에 대한 **정확한 이해**
  - 기능의 **효율적이고 적시적인 구현**
  - **버그를 최소화**하는 높은 코드 품질
  - 유연한 변경 사항 대응을 위한 **재사용 및 확장 가능한 애플리케이션** 구조 설계
  - 부하 급증에 대응 가능한 **높은 안정성**
  - 협업을 위한 **코드 및 문서 가독성**
- 반드시 사용되어야 하는 기술 스택
  - Java, Spring, JPA
  - 그 외엔 필요하면 사용 가능
- H2 데이터베이스를 사용할 것
- 요구사항에 정의되지 않았어도 있으면 좋을 것 같은 기능이 있다면 추가적으로 구현 가능
- 과제 평가 기준
  - 애플리케이션이 잘 실행되는가?
  - 모든 요구사항이 구현되었는가?
  - 예외 상황이 잘 고려되었는가?
  - 설계가 간결하고 효율적인가?
  - API 스펙이 명확한가?
  - 테스트 코드가 작성되어 있는가?
  - 문서화가 잘 되었는가?
  - 추가 사항을 구현했는가?

- 구현해볼 추가 사항
  - [x] 이메일 유효성 검증
  - [x] 패스워드 인코딩 기능
  - [x] 패스워드 변경 기능
  - [x] 휴대폰 번호 변경 기능
  - [x] 회원 유형 변경 기능

---

이 애플리케이션은 회원으로 등록된 사용자만 강의를 수강하거나 등록할 수 있다.

- [x] 회원 가입 화면에서 필요한 정보를 입력하고 가입하기 버튼을 클릭하면 가입이 완료된다.
- [x] 가입 시 입력하는 회원 정보는 다음과 같다.
  - [x] 이름: 정해진 규칙은 없지만 최소 3자 이상의 유효성 검증이 필요해보인다.
  - [x] 이메일: 이메일도 정해진 규칙은 없지만 이메일 규격에 맞게 입력받아야 할 것 같다.
  - [x] 휴대폰 번호: 휴대폰 번호도 정해진 규칙은 없지만 한국 휴대폰 번호 정규식이 필요해 보인다. 
  - [x] 비밀번호
    - [x] 최소 6자 이상 10자 이하
    - [x] 영문 소문자, 대문자, 숫자 중 최소 두 가지 이상 조합이 필요하다
  - [x] 회원 유형: 수강생 / 강사

이 애플리케이션은 강의를 개설할 수 있다.

- [x] 강사로 가입한 회원이 자신의 강좌를 등록할 수 있다.
- [x] 등록된 강좌는 수강생 회원이 신청할 수 있다.
- [x] 강의 등록 화면에서 강의 정보를 입력하고, 등록하기 버튼을 클릭하면 등록이 완료된다.
- [x] 수강생 회원은 강의를 등록할 수 없다.
- [x] 강의 등록 시 입력하는 정보는 아래와 같다.
  - [x] 강의명
  - [x] 최대 수강 인원
  - [x] 가격

이 애플리케이션은 강의를 수강 신청할 수 있다.

- [x] 수강생 회원이 강의 목록을 확인하고 신청하는 기능이다.
- [x] 웹의 강의 목록에서 수강하고 싶은 강의를 선택하여 신청한다.
- [x] 강의 목록은 페이징 처리가 필요하다. 1페이지당 20개의 강의 목록을 불러온다.
- [x] 최근 등록순, 신청자 많은 순, 신청률 높은 순으로 정렬이 가능하다.
- [x] 신청자 많은 순은 현재 기준 신청자가 많은 강의 순으로 표시한다.
- [x] 신청률 높은 순(신청률 = 신청자 수/최대 수강 인원 수)으로 계산하여 비율이 높은 강의부터 표시한다.
- [x] 강의 목록의 각 강의 아이템은 아래 정보를 표시한다.
  - [x] 강의명
  - [x] 가격
  - [x] 강사명
  - [x] 현재 신청자 수
  - [x] 최대 수강 인원 수
- [x] 강의는 여러 개를 선택하여 한 번에 신청할 수 있다.
- [x] 강사 회원도 수강 신청이 가능하다.
- [x] 인기 강좌는 강의가 등록되자마자 신청이 폭주할 수 있으므로 많은 사람이 동시에 신청하더라도 설정된 최대 수강 인원만큼만 선착순으로 신청 처리 되어야 한다.

# 도메인 모델

## 회원(Member)
_Entity_

### 속성
- `id`: ID
- `name`: 이름
- `email`: 이메일
- `phoneNumber`: 핸드폰 번호
- `encodedPassword`: 비밀번호 (해시)
- `role`: 회원 유형

### 행위
- `static register()`: 회원을 등록한다
- `changePassword()`: 비밀번호를 변경한다
- `changePhoneNumber()`: 핸드폰 번호를 변경한다
- `changeRole()`: 회원 유형을 변경한다
- `verifyPassword()`: 비밀번호를 검증한다
- `isInstructor()`: 회원 유형이 강사인지 확인한다

## 회원 유형 (MemberRole)
_Enum_
### 상수
- `INSTRUCTOR`: 강사
- `STUDENT`: 수강생

## Email
_Value Object_
### 속성
- address : 이메일 주소

## PhoneNumber
_Value Object_
### 속성
- value : 핸드폰 번호

## 비밀번호 인코더 (PasswordEncoder)
_Domain Service_
### 행위
- `encode()`: 비밀번호 인코딩
- `matches()`: 비밀번호 일치 여부 확인

---

## 강의(Course)
_Entity_

### 속성
- `id`: ID
- `title`: 강의명
- `capacity`: 최대 신청 인원
- `price`: 가격
- `enrollCount`: 신청자 수 (역정규화)
- `instructor`: 강사

### 행위
- `static open()`: 강의를 등록한다
- `increaseEnrollCount()`: 신청자 수를 하나 증가한다

---

## 수강신청(Enrollment)

### 속성
- `id`: ID
- `applicant`: 수강신청자
- `course`: 강의
- `enrolledAt`: 수강 신청 기록

### 행위
- `apply()`: 수강을 신청한다
